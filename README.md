# SQL Practice Repo

This repository contains SQL exercises and datasets used to develop and reinforce core querying skills.

The focus is on building a strong foundation in SQL by progressing from simple queries to more complex data analysis and transformations.

This repository demonstrates my progression in SQL, from basic querying to more structured and analytical use cases, and supports my development towards a data engineering role.

This SQL practice work forms the foundation for more structured projects, such as:

- Data cleaning and transformation pipelines  
- Relational modelling  
- Data analysis and reporting  

---

## Purpose

The aim of this repository is to practise and consolidate key SQL concepts, including:

- Filtering and sorting data  
- Aggregation and grouping logic  
- Working with multiple tables and relationships  
- Translating business-style questions into SQL queries  

This work supports my broader projects by strengthening the fundamentals needed for data analysis and data engineering tasks.

---

## Skills Demonstrated

- **Core queries:** `SELECT`, `WHERE`, `ORDER BY`  
- **Aggregation:** `SUM`, `AVG`, `COUNT`, `MIN`, `MAX`  
- **Grouping & filtering:** `GROUP BY`, `HAVING`  
- **Joins:** `INNER JOIN`, `LEFT JOIN`, multi-table joins  
- **Set operations:** `UNION`, `UNION ALL`  
- **Views:** creating reusable query outputs  
- **Data modelling basics:** table creation, primary keys, foreign keys  

---

## Repository Structure

### Books Dataset

- Practice queries on a single-table dataset  
- Focus on:
  - Filtering (e.g. genre, year)  
  - Aggregations (e.g. total sales, average rating)  
  - Business questions (e.g. top-performing genres, revenue)  

---

### Sales / Customers Dataset

Multi-table relational dataset including:

  - `SalesPerson`
  - `Customer`
  - `Sale`

Practises:

  - Joins across multiple tables  
  - Sales analysis by customer and salesperson  
  - Understanding table relationships  

---

### Joins, Unions & Views

Focused exercises covering:

  - `INNER JOIN` vs `LEFT JOIN`  
  - Combining datasets using `UNION` / `UNION ALL`  
  - Creating and querying `VIEW`s  

Includes examples of:

  - Aggregation with joins  
  - Filtering grouped results using `HAVING`  

---

## Example Questions Explored

- Which book genre generates the most revenue?  
- Who is the top-selling author?  
- Which customers have the highest total spend?  
- Which salespeople exceed performance thresholds?  
- How do we combine datasets with different structures?  

---

## Notes

- This repository is focused on **practice and learning**, not production-ready solutions  
- Queries are intentionally varied to build familiarity with different SQL patterns  
- Emphasis is placed on understanding *why* a query works, not just writing it
